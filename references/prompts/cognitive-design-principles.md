# Cognitive Design Principles for Presentations

Shared reference for all PPT generation prompts. Apply these evidence-based principles to maximize audience comprehension and retention.

## Miller's Law (Working Memory Capacity)

- Human working memory holds **4 ± 1 chunks** in active focus (Cowan, 2001; tighter than Miller's original 7±2)
- **Slide rule**: Maximum 5 distinct information units per slide. Group related items visually to form single chunks.
- Use the **one-idea-per-slide** principle for complex topics
- Each card in a Bento Grid counts as 1 chunk if visually cohesive

## Mayer's Multimedia Learning Principles

The three highest-impact principles for static slides (with measured effect sizes):

| Principle | Rule | Effect Size |
|-----------|------|-------------|
| **Spatial Contiguity** | Place labels/text NEAR the corresponding graphic, not separated | 1.10 (highest) |
| **Coherence** | Remove ALL extraneous material — decorative images, tangential text, unnecessary animations | 0.86 |
| **Signaling** | Use visual cues (bold, color, size, arrows) to guide attention to key structure | 0.41 |

Additional applicable principles:
- **Segmenting**: Break complex content across multiple slides rather than overloading one
- **Multimedia**: Use words + graphics together, not words alone
- **Redundancy**: Do NOT put full narration text on-screen (for live presentations)

## Gestalt Principles for Slide Layout

| Principle | Application in Slides |
|-----------|----------------------|
| **Proximity** | Related cards/elements must be adjacent; use whitespace as separator between unrelated groups |
| **Similarity** | Same-level items share visual treatment (color, size, shape) consistently |
| **Continuation** | Align elements on grid lines; the eye follows implied alignment paths |
| **Figure-Ground** | Primary content must have sufficient contrast against background; avoid ambiguous layering |
| **Common Region** | Use card boundaries (Bento Grid) to group related information |

## Cognitive Load Theory (Sweller)

Three types of cognitive load — design decisions affect extraneous load:
- **Intrinsic load**: Content complexity — reduce by segmenting and sequencing
- **Extraneous load**: Imposed by poor design (clutter, inconsistency, noise) — **eliminate aggressively**
- **Germane load**: Mental effort that builds understanding — maximize through structure and examples

**Design target**: Minimize extraneous load so cognitive budget is available for the actual message.

## Visual Hierarchy: 3-Second Test

For each slide, apply the 3-second test:
1. Cover your eyes, look at the slide for 3 seconds, look away
2. What did you see first? Second? Third?
3. If the answer doesn't match intended hierarchy → redesign
4. If you couldn't identify the key message → reduce content density

## Three-Tier Hierarchy Model

1. **Primary** (title, hero number, key message): largest, highest contrast, most whitespace around it
2. **Secondary** (supporting data, subheads): medium size, moderate contrast
3. **Tertiary** (source citations, footnotes, labels): smallest, lowest contrast, 50-70% opacity

Never use more than 3 hierarchy levels on a single slide.
